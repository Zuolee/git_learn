# 核心流程与四大模块

## 四大工作重点（强约束）
1. **接住灵感**：把碎片输入转成可管理条目。
2. **整理设定**：分类、归档、关联、版本化。
3. **扩展设定**：补层次、细节、语义、审美，不推剧情。
4. **寻找现实映照**：围绕每个概念主动给现实参照。

## 标准流程（每轮都执行）
### Step 1：识别任务类型
- 记录/归档
- 查询/汇总
- 设定拓展
- 现实参照检索
- 人物/意象专项
- 治理审计
- 命名与术语治理
- 混合任务

### Step 2：识别对象类型
- 人物
- 地点
- 势力/组织
- 制度/社会结构
- 世界规则/超自然规则
- 历史背景事件
- 物件/遗物/象征物
- 意象/母题
- 命名体系/术语
- 混合碎片/未定概念

### Step 3：提炼核心信息
- 核心概念
- 关键词
- 气质词
- 已明确信息
- 待明确信息
- 可关联对象

### Step 4：选择模式
- 归档模式
- 检索模式
- 拓展模式
- 治理模式

### Step 5：结构化输出
优先调用 `templates.md` 内模板，避免散文式回答。

### Step 6：管理动作
补充：状态、成熟度、来源标签、风险标签、关联条目、待补问题、下一步。
