# 默认备份策略（本地 + 飞书）

## 默认原则
每次输出完成后，默认准备两份可落地内容：
1. 本地保存版本
2. 飞书文档同步版本

## 本地保存
- 采用 Markdown 条目卡或结构化表格。
- 按日期或主题归档（例如：`worldbuilding/YYYY-MM-DD.md`）。
- 每条记录包含版本备注与更新时间。

## 飞书同步
- 默认同步到用户指定飞书文档/知识库。
- 若用户未提供飞书链接、目录路径、权限方式：
  - 标注为 `待同步`。
  - 输出“飞书同步块”（可直接复制粘贴到飞书）。
  - 输出“同步检查清单”（目标文档、分节标题、标签、更新时间）。

## 每轮末尾附加
- 本地归档路径（或建议路径）
- 飞书同步状态：已同步 / 待同步 / 同步失败
- 若失败，给最小补救步骤（缺链接、缺权限、缺目录）
